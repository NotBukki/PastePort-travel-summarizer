# PastePort
A travel document parser that extracts structured trip events, budgets, and currency info from pasted text using an LLM backend; built with React + Vite (client) and Node.js/Express (server).

## Env & setup
- Copy `.env.example` to `.env` in the repo root before running — the server reads env vars for the LLM API key
- Three separate `package.json` files exist (root, `client/`, `server/`) — run `npm install` in each before starting
- An external LLM API (configured via env vars) must be reachable; the server will fail silently on parse requests if the key is missing or wrong

## Architecture
- `client/src/components/` — React UI components: paste input, timeline, event cards, budget panel, suggestions panel, currency switcher, loading overlay, onboarding tour
- `client/src/hooks/` — custom React hooks: `useTripParser` orchestrates parse requests; `useCurrency` manages currency state and conversion; `useConsent` manages data-use consent state
- `client/src/` — Vite/React app entry, global CSS
- `server/routes/` — Express route handlers; `parse.js` is the only API route, handling POST requests from the client
- `server/prompts/` — LLM prompt templates; `extractor.js` constructs the extraction prompt sent to the LLM
- `server/utils/` — server-side utilities; `maskPii.js` strips sensitive data from text before it reaches the LLM
- `server/` — Express app bootstrap and middleware setup

## Entry points
- `client/src/main.jsx` — mounts the React app to the DOM
- `client/src/App.jsx` — top-level component, wires together all panels and hooks
- `server/index.js` — starts the Express server and registers routes
- `server/routes/parse.js` — defines `POST /parse`, the single API endpoint the client calls; applies PII masking before the prompt is built
- `server/prompts/extractor.js` — controls the LLM prompt structure; changes here affect all parsing behavior
- `server/utils/maskPii.js` — regex-based PII scrubber; edit here to add or adjust masking rules

## Conventions
- **File naming** — React components use PascalCase filenames and default exports (e.g., `EventCard.jsx`); hooks use camelCase prefixed with `use` (e.g., `useCurrency.js`)
- **Imports** — client code uses relative imports; no barrel `index.js` files observed in components or hooks — import each file directly
- **Data patterns** — parsed trip data flows from `useTripParser` down through props; currency conversion is isolated in `useCurrency` and not mixed into parse logic

## Gotchas & footguns
- The root `package.json` is a workspace/orchestration file — do not add application dependencies there; add client deps under `client/` and server deps under `server/`
- `ANTHROPIC_API_KEY` must be set in `.env` — the server uses Claude Haiku 4.5 via `@anthropic-ai/sdk`; missing key causes `/parse` to return an error with no client-side warning beyond a failed fetch
- `client/public/favicon.svg` and build output in `client/dist/` (generated by Vite) must never be manually edited
- The client dev server proxies API requests to the Express server — check `client/vite.config.js` for the proxy target port before changing `server/index.js` port
- The Vite proxy has a custom `error` handler (`configure` hook in `vite.config.js`) that returns a JSON 503 when Express hasn't started yet — do not remove it or first-parse cold-start failures will resurface as a client-side SyntaxError
- The onboarding tour (`OnboardingTour.jsx`) targets elements by stable DOM `id` attributes set in `PasteInput.jsx` (`segment-text-first`, `segment-mode-toggle-first`, `traveler-type-selector`) and `App.jsx` (`btn-parse-trip`). If those IDs are renamed or removed the tour steps will silently skip. Tour completion is stored under the `pp_tour_v1` localStorage key — remove it in DevTools to replay the tour during development
- PII masking applies only to text segments in `parse.js`; image and PDF uploads reach the LLM as binary content and are not masked by regex

## Commands
- `npm run dev` — starts both client (Vite) and server together if a root-level dev script exists; otherwise run separately:
  - `cd client && npm run dev` — Vite dev server
  - `cd server && npm run dev` or `node index.js` — Express server
- `cd client && npm run build` — compiles React app to `client/dist/`
- `cd client && npm run preview` — serves the production build locally
